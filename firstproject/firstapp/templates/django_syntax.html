Django Tags

In Django templates, you can perform programming logic like executing if statements and for loops. These keywords, if and for, are called "template tags" in Django. To execute template tags, we surround them in '{ % % }' brackets.

{% with greeting=1 %} {% if greeting == 1 %}

Hello! {{ user }}. Greeting is '1'.

{% else %}

Bye! {{ user }}. Greeting is not '1'.

{% endif %} {% endwith %}

Django If Else

If Statement

{% if greeting == 1 %}

Hello! {{ user }}. Greeting is '1'.

{% endif %}

Elif

{% if greeting == 1 %}

Welcome! {{ user }}. Greeting is '1'.

{% elif greeting == 2 %}

Hello! {{ user }}. Greeting is '2'.

{% endif %}

Else

{% if greeting == 1 %}

Goodbye! {{ user }}. Greeting is '1'.

{% elif greeting == 2 %}

Hello! {{ user }}. Greeting is '2'.

{% else %}

Welcome! {{ user }}

{% endif %}

Operators : The above examples uses the == operator, which is used to check if a variable is equal to a value, but there are many other operators you can use, or you can even drop the operator if you just want to check if a variable is not empty.

{% if greeting %}

Hello! {{ user }}. If greeting...

{% endif %}

== : Is equal to

{% if greeting == 1 %}

Hello! {{ user }}. Greeting is '1'.

{% endif %}

!= : Is not equal to

{% if greeting != 2 %}

Hello! {{ user }}. Greeting is not equal to '2'.

{% endif %}

< : Is less than

{% if greeting < 3 %}

Hello! {{ user }}. Greeting is less than '3'.

{% endif %}

<= : Is less than, or equal to

{% if greeting <= 3 %}

Hello! {{ user }}. Greeting is less than or equal to '3'.

{% endif %}

> : Is greater than

{% if greeting > 0 %}

Hello! {{ user }}. Greeting is greater than '0'.

{% endif %}

>= : Is greater than, or equal to

{% if greeting >= 1 %}

Hello! {{ user }}. Greeting is greater than or equal to '1'.

{% endif %}

and : To check if more than one condition is true

{% if greeting == 1 and day == "Friday" %}

Hello Weekend! Greeting is '1' and day is 'Friday'.

{% endif %}

or : To check if one of the conditions is true

{% if greeting == 1 or greeting == 5 %}

Greeting is either 1 or 5.

{% endif %}

and/or : Combine and and or

{% comment %} {% if greeting == 1 and day == "Friday" or greeting == 5 %} {% endcomment %}

in : To check if a certain item is present in an object

{% if 'Banana' in fruits %}

Hello! If banana is in fruits.

{% else %}

Goodbye! If banana is not in fruits.

{% endif %}

not in : To check if a certain item is not present in an object

{% if 'Banana' not in fruits %}

Hello! If banana is not in fruits.

{% else %}

Goodbye! If banana is in fruits.

{% endif %}

is : Check if two objects are the same

This operator is different from the == operator, because the == operator checks the values of two objects, but the is operator checks the identity of two objects.

{% if x is y %}

YES, If 'x' is 'y'.

{% else %}

NO, If 'x' is not 'y'.

{% endif %} {% if x == y %}

YES, If 'x == y'.

{% else %}

NO, If 'x != y'.

{% endif %} {% with var1=x var2=x %} {% if var1 is var2 %}

YES, If 'var1' is 'var2'.

{% else %}

NO, If 'var1' is not 'var2'.

{% endif %} {% endwith %}

is not : To check if two objects are not the same

{% if x is not y %}

YES, If 'x' is not 'y'.

{% else %}

NO, If 'x' is 'y'.

Django For Loop

A 'for' loop is used for iterating over a sequence, like looping over items in an array, a list, or a dictionary.

Fruits in the List are:

    {% for fruit in fruits %}
  • {{ fruit }}
    • {% endfor %}

Loop through a list of dictionaries:

{% for car in cars %} {% endfor %}
Brand Model Year
{{ car.brand }} {{ car.model }} {{ car.year }}

Data From a Model

{% for member in mymembers %} {% endfor %}
id firstname lastname phone joined_date
{{ member.id }} {{ member.firstname }} {{ member.lastname }} {{ member.phone }} {{ member.joined_date }}

Reversed

The reversed keyword is used when you want to do the loop in reversed order.

{% for member in mymembers reversed %} {% endfor %}
id firstname lastname phone joined_date
{{ member.id }} {{ member.firstname }} {{ member.lastname }} {{ member.phone }} {{ member.joined_date }}

Empty

The empty keyword can be used if you want to do something special if the object is empty. The empty keyword can also be used if the object does not exist.

Members in emptytestobject

    {% for x in emptytestobject %}
  • {{ x.firstname }}
    • {% empty %}
  • No members
    • {% endfor %}

Members in myobject

    {% for x in myobject %}
  • {{ x.firstname }}
    • {% empty %}
  • No members
    • {% endfor %}

Loop Variables

1. forloop.counter

The current iteration, starting at 1.

Count of fruit in fruits list

    {% for x in fruits %}
  • {{ forloop.counter }}
    • {% endfor %}

2. forloop.counter0

The current iteration, starting at 0.

Count of fruit in fruits list

    {% for x in fruits %}
  • {{ forloop.counter0 }}
    • {% endfor %}

3. forloop.first

Allows you to test if the loop is on its first iteration.

Example: Draw a blue background for the first iteration of the loop.

    {% for x in fruits %}
  • {{ x }}
    • {% endfor %}

4. forloop.last

Allows you to test if the loop is on its last iteration.

Example: Draw a blue background for the last iteration of the loop.

    {% for x in fruits %}
  • {{ x }}
    • {% endfor %}

5. forloop.revcounter

The current iteration if you start at the end and count backwards, ending up at 1.
    {% for x in fruits %}
  • {{ forloop.revcounter }}
    • {% endfor %}

6. forloop.revcounter0

The current iteration if you start at the end and count backwards, ending up at 0.
    {% for x in fruits %}
  • {{ forloop.revcounter0 }}
    • {% endfor %}

Django Comment

Comments allows you to have sections of code that should be ignored.

Welcome Everyone

{% comment %}

Welcome ladies and gentlemen

{% endcomment %}

You can add a message to your comment, to help you remember why you wrote the comment, or as message to other people reading the code.

Add a description to your comment.

Welcome Everyone

{% comment "this was the original welcome message" %}

Welcome ladies and gentlemen

{% endcomment %}

Smaller Comments

You can also use the {# ... #} tags when commenting out code, which can be easier for smaller comments

Comment out the word Everyone:

Welcome{# Everyone#}
